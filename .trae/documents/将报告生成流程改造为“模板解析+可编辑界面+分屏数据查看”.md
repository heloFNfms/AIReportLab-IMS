## 目标
- 将当前“模板选择 → 模板分析 → 数据选择 → AI生成报告”的四阶段流程，改造为“模板选择 → 解析 → 可编辑界面（导入解析结果） → 分屏数据查看与人工撰写”。
- 暂时不调用AI生成，保留相关能力但通过特性开关隐藏/禁用。
- 提供可用的编辑体验与基础预览能力（JSON/CSV/TXT/部分DOCX文本）。

## 现状确认
- 前端：Vue3 + Vite + Element Plus + Pinia + Router，核心页面 `frontend/src/views/Dashboard.vue`（步骤控制与四阶段向导）。
  - 步骤状态 `generatorStep`（Dashboard.vue:288, 297），模板/数据文件选择与分析/生成触发在此页面集中。
- 后端：FastAPI 分层，模板分析 `backend/app/api/endpoints/templates.py` 调用 `backend/app/services/ai/template_analyzer.py`；生成 `backend/app/api/endpoints/reports.py` 调用 `backend/app/services/ai/report_generator.py`。

## 前端改造
- 引入新的“编辑器模式”页面或在现有 `Dashboard.vue` 中新增一个“编辑器”视图：
  - 方案A（增路由，推荐）：新增 `frontend/src/views/Editor.vue`，路由 `/editor/:templateId`。
  - 方案B（沿用Dashboard）：在 Dashboard 增加 `generatorStep === 3` 为编辑器视图，并将原“生成报告”视图移除或隐藏。
- 布局：使用 `el-container` + `el-aside`/`el-main` 或 `el-row`/`el-col` 实现分屏：
  - 左侧：文章编辑器（先用 `el-input type="textarea"` 保证零依赖稳定可用，后续可选集成 Monaco/Tiptap）。
  - 右侧：数据文件查看器（文件树 + 预览区域，支持类型化渲染）。
- 入口流转：
  - 在模板选择并完成解析后（Dashboard.vue:771–789 调用 `analyzeTemplate`），跳转到 `/editor/:templateId` 并将解析结果通过路由参数或 Pinia 存储传递。
- 解析结果导入编辑器：
  - 将 `template_analyzer` 返回的结构化信息（章节/风格/格式规则/数据要求）转换为 Markdown 纲要（标题、占位段落、TODO 提示）。
  - 在 Editor 加载时初始化编辑器内容为该 Markdown 纲要。
- 功能开关：
  - 在前端配置新增 `featureFlags.disableAIGeneration = true`（可读取后端 `/config` 或 `.env`），据此隐藏 Dashboard 中的“生成报告”按钮与对应步骤视图（Dashboard.vue:796–823 附近）。
- 草稿与自动保存：
  - 在 Editor 中监听内容变更，debounce 后调用草稿保存接口；支持手动“保存草稿”、“导出 Markdown”。

## 数据查看与分屏
- 右侧数据文件列表来源沿用 `frontend/src/stores/file.ts` 与 `frontend/src/api/files.ts`：
  - 文件树或列表（按已上传文件分类）。
  - 打开文件时调用新建的“预览接口”，返回不同类型的内容：
    - `application/json`：前端 `JSON.stringify(value, null, 2)` 高亮显示。
    - `text/csv`：用 Element Plus 的 `el-table` 简单呈现（按逗号分隔，首行表头）。
    - `text/plain`：`<pre>` 或 `<el-scrollbar>` 原样显示。
    - `application/vnd.openxmlformats-officedocument.wordprocessingml.document`：调用后端的 `extract_text_from_docx` 返回纯文本预览。
- 支持分屏宽度拖拽（Element Plus 样式 + `resize: horizontal` 或引入简易分割条，初期可固定比例 50/50，再迭代可拖拽）。

## 后端改造
- 新建“文件预览”接口：`GET /files/{file_id}/preview`
  - 读取文件（本地或 OSS 签名 URL），返回 `{ mime_type, content, preview_type }`：
    - `preview_type` 枚举：`json` | `csv` | `text`。
  - DOCX 使用现有 `template_analyzer.extract_text_from_docx()` 产出纯文本预览。
- 草稿接口：
  - `POST /drafts` 保存草稿：入参包含 `template_id`、`content`、`format='markdown'`、`title`。
  - `GET /drafts/{template_id}` 加载最近草稿。
  - 存储可采用本地 `backend/uploads/drafts/{user}/{template_id}.md` 或数据库（若已有 `reports` 表可添加 `is_draft` 标记）。
- 配置接口：
  - `GET /config` 返回前端特性开关（如 `disableAIGeneration`）。
- 保留并禁用 AI 生成：
  - 不改动 `POST /reports/generate` 的服务逻辑，仅在前端默认不调用；后端可基于配置校验并返回 403，以免误触。

## 解析结果到 Markdown 纲要的转换
- 在前端新增一个转换工具函数 `src/utils/templateToOutline.ts`：
  - 输入：模板分析结构（报告类型、章节列表、每章要点、格式要求、数据要求）。
  - 输出：Markdown 文本，形如：
    - `# 报告标题（待填写）`
    - `## 摘要（建议 200–300 字）`
    - `## 第一章：章节名` + 要点列表转为子标题/项目符号 + `TODO(数据要求)` 注释。
  - 可添加 Front Matter（YAML）记录元信息（模板 ID、日期、作者）。

## 交互与体验优化建议
- 编辑器：
  - 初期使用 `el-input` 文本域；后续可选集成 Monaco（代码/Markdown 高亮）或 Tiptap（所见即所得）。
  - 提供“插入数据片段”按钮，可从右侧预览区域复制选区插入到左侧文档。
- 草稿：
  - 自动保存（每 5 秒或停止输入 1 秒），支持版本历史（保存 N 版轮换）。
- 数据预览：
  - 大文件分页/懒加载，避免卡顿。
- 安全：
  - 过滤潜在脚本注入，预览区域采用纯文本渲染。
